# Memory

This directory documents the future memory model for EPDC Site Factory. No memory implementation exists yet.

## Working Memory

Working memory is intended to hold short-lived execution context for an active generation run, such as intermediate planning notes, task state, and temporary coordination data.

## Long-Term Memory

Long-term memory is intended to preserve reusable knowledge across runs, such as template learnings, common implementation patterns, historical decisions, and successful project examples.

## Future SQLite Storage

A future version of this repository may use SQLite as a lightweight local storage layer for structured memory records, execution traces, specifications, and project metadata.

## Future Vector Search Support

A future version of this repository may add vector search support for semantic retrieval across specifications, skills, template knowledge, and prior generated project artifacts.

This phase provides documentation only.
